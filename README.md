# Physical AI & Humanoid Robotics Course

This repository contains the complete course materials for the "Physical AI & Humanoid Robotics Course". The course is designed to teach students, makers, and engineers about embodied intelligence in robotics, focusing on humanoid robots.

## Course Structure

The course is divided into four core modules:

### Module 1: ROS 2 - The Robotic Nervous System
- Introduction to ROS 2 and Middleware Concepts
- ROS 2 Nodes, Topics, and Services
- Bridging Python Agents to ROS Controllers (rclpy)
- URDF and Humanoid Robot Models

### Module 2: Gazebo & Unity - The Digital Twin
- Physics Simulation Basics
- Sensors and Environment Building
- High-Fidelity Rendering & Human-Robot Interaction
- Integrating Gazebo and Unity

### Module 3: NVIDIA Isaac - The AI-Robot Brain
- Isaac Sim & Synthetic Data Generation
- Isaac ROS and VSLAM Navigation
- Nav2 Path Planning for Bipedal Robots
- Advanced AI-Robot Brain Techniques

### Module 4: Vision-Language-Action (VLA)
- Voice-to-Action with Whisper
- Cognitive Planning using LLMs
- Integrating Vision, Language, and Action
- Capstone: Autonomous Humanoid

## Course Features

- Comprehensive educational content for humanoid robotics
- Step-by-step tutorials with code snippets, simulations, and diagrams
- Integration with modern AI tools (LLMs, Whisper, Isaac Sim)
- RAG-based chatbot for answering questions from course content
- Hands-on exercises and projects

## Prerequisites

- Basic understanding of programming concepts (Python preferred)
- Familiarity with Linux command line
- Understanding of fundamental mathematics (linear algebra, calculus)
- Interest in robotics and AI systems

## Getting Started

1. Clone the repository
2. Follow the course modules in order (start with Module 1)
3. Complete the exercises and hands-on projects
4. Use the integrated RAG chatbot for additional support

## Course Deployment

This course is deployed as a Docusaurus website with embedded RAG chatbot functionality. The content is structured to be incrementally deployable after each module completion.
